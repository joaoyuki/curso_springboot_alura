Aula 02 - Por que autenticar via Token


Em uma aplicação web tradicional, a autenticação é feita via Session:



Sempre que um usuário faz um requisição para o servidor pela primeira vez, é criada uma JSESSIONID na memória, identificando aquele usuário. Esse JSESSIONID é devolvido como uma resposta para o browser, que armazena isso como um cookie, e esse cookie será utilizado nas próximas requisições feitas pelo usuário.
Esse modelo tem os seguintes problemas:
• Para cada usuário, teremos um espaço em memória na máquina, podendo ficar muito custoso
• Caso a máquina caia, iremos perder todos esses id's
• Não conseguimos escalar os servidores, pois o id fica apenas naquela máquina, ou teríamos de copiar todos os id'd em todas as máquinas
